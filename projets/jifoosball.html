Projet Tournois de Babyfoot : Application Web et Mobile

Contexte et Objectifs

Dans le cadre du semestre 5 du BUT Informatique, nous avons lancé un projet ambitieux : développer une application web et mobile pour gérer des tournois de babyfoot. Contrairement aux projets académiques classiques, ce projet est réalisé pour des clients extérieurs, avec une vision concrète et orientée utilisateur.

Les objectifs principaux sont :

  • Permettre l’organisation de tournois avec phases de poules et brackets.
  • Gérer des équipes fixes ou aléatoires pour chaque match.
  • Implémenter un système de classement basé sur le Elo.
  • Afficher les bars partenaires sur une carte interactive pour faciliter la localisation des tournois.
  • Créer une application réactive et intuitive, accessible sur web et mobile.

Architecture et Choix Techniques

Pour répondre aux besoins du projet, nous avons pris plusieurs décisions techniques clés :

  • API : Développée en Kotlin pour profiter des fonctionnalités modernes et typées, permettant une communication fluide avec le web et le mobile.
  • Front-end Web : Réalisé en React avec Vite pour un rendu dynamique, une compilation rapide et une grande flexibilité dans le développement.
  • Front-end Mobile : Développé en React Native avec Expo, permettant de tester rapidement l’application sur téléphone via Expo Go et de partager une partie de la logique avec le web.
  • Base de données : MySQL, conceptualisée pour gérer utilisateurs, équipes, tournois, matches et scores.

Cette architecture sépare clairement le web et le mobile, tout en utilisant une API commune, ce qui simplifie l’évolution future et la maintenance.

Fonctionnalités Principales (en cours)

  • Création et gestion de tournois avec phases de poules et brackets.
  • Gestion d’équipes fixes ou aléatoires pour chaque match.
  • Système de classement Elo dynamique pour refléter les performances des joueurs et équipes.
  • Carte interactive affichant les bars partenaires et les lieux de tournois.
  • Interface utilisateur responsive et adaptée pour web et mobile.

Défis et Apprentissages

Ce projet a présenté plusieurs défis particuliers par rapport aux projets académiques :

  • Apprentissage de Kotlin : Nouveau langage pour moi, avec des concepts différents de PHP et JavaScript.
  • Découverte de React Native : Intégration d’un framework mobile moderne, très différent de React Web, et adaptation des composants pour un rendu natif.
  • Architecture multi-plateforme : Maintenir une API unique pour deux clients (web et mobile) tout en assurant cohérence et performances.
  • Complexité des tournois : La gestion des phases de poules, brackets et équipes aléatoires nécessite une logique métier solide et modulable.
  • Expérience utilisateur : Concevoir une interface claire et intuitive malgré la diversité des fonctionnalités et des supports.

Compétences Développées

  • Maîtrise initiale de Kotlin pour le développement d’API backend.
  • Création d’interfaces web et mobiles avec React et React Native.
  • Conception d’une architecture scalable et multi-plateforme.
  • Gestion d’un projet complexe avec un produit concret pour des clients externes.
  • Renforcement des compétences en travail d’équipe, communication et adaptation à de nouveaux outils.

Conclusion et Perspectives

Bien que le projet soit encore en cours, JiFoosball constitue une expérience riche et formatrice. La complexité du projet, combinée à l’apprentissage de nouvelles technologies et frameworks, m’a permis de développer des compétences concrètes pour des projets professionnels multi-plateformes. À terme, l’application permettra aux utilisateurs de gérer leurs tournois facilement, avec une interface fluide et un système de classement fiable.
